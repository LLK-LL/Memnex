## What this solves

Since 2024, CNKI (China National Knowledge Infrastructure 鈥?the dominant Chinese academic database) has added AIGC detection on top of its existing plagiarism check. All Chinese theses and journal submissions must pass this check. **Even human-written papers often get flagged**, because academic prose naturally resembles LLM output.

Most "AI rate reduction" advice (synonym swaps, padding) doesn't work. This skill is distilled from a real case study:

| Metric | First check | After rewrite | Change |
|--------|-------------|---------------|--------|
| Overall AI rate | 20.6% | **10.1%** | -10.5pp |
| Section 1 | 4.2% | **0.0%** | Cleared |
| Section 2 | 26.0% | **15.8%** | -10.2pp |
| Section 3 | 22.6% | **13.7%** | -8.9pp |

More importantly: **all "red highlighted" passages downgraded to "suspected"**. ~10% is roughly the floor for Chinese academic prose; going lower sacrifices academic register.

## Core insight: what the detector actually catches

It's not catching "AI-written" text 鈥?it's catching 5 linguistic patterns:

1. **Predictable syntactic rhythm** 鈥?LLM sentence length clusters at 20鈥?5 chars (normal); humans follow a Zipfian distribution
2. **Uniform information density** 鈥?LLMs produce evenly dense paragraphs; humans alternate dense argumentation with looser exposition
3. **Fixed syntactic role of technical terms** 鈥?LLMs put terms in subject position; humans more often use them as topic or object (**don't change the term, change its role**)
4. **Functionally redundant connectives** 鈥?LLMs overuse "鍥犳/浠庤€?杩涜€?鐢辨" (all marking causal progression); humans use connectives with varied functions
5. **Functionally equivalent template paragraphs** 鈥?three/four-way parallel structures, abstract noun chains, meta-discourse closings

## Usage

### Option 1: Load as Claude Skill

1. Copy `SKILL.md` into your Claude Skills directory
2. In a Claude chat, type `/cnki-aigc-闄嶉噸` or describe the task
3. Paste a flagged paragraph (<800 chars per turn recommended)
4. Claude returns: rewritten version + which features were removed + what stayed untouched

### Option 2: Use as a system prompt

Paste the body of `SKILL.md` as a system prompt, then submit your paragraph.

### Input-length routing

- **<800 chars**: interactive mode (one paragraph at a time)
- **>800 chars**: batch mode (lists high-risk paragraphs first, then rewrites selected ones)

## Out of scope

- Abstracts (structurally template-like; usually not counted toward AI rate anyway)
- Reference-heavy paragraphs ("X et al. (2020) found...")
- Standardized methods sections ("we use two-way fixed effects")
- Non-academic text

---

## What NOT to do

- Don't swap synonyms 鈥?the detector tracks syntax, not vocabulary
- Don't go fully colloquial 鈥?that just shifts from "AI tone" to "blog tone"
- Don't strip all hedge words 鈥?hedges are a human signal
- Don't put quotes around terms to create distance 鈥?classic LLM tic
